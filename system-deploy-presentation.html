From artifact to prospect-ready page.

The workflow removes the manual deployment and Webflow handoff that made custom HTML presentations difficult for the sales team to use.

01Create the artifact

The sales team creates a self-contained HTML presentation using branded design skills.

02Publish through Vercel

The skill calls the Vercel API and publishes the HTML presentation into the corporate Vercel account.

03Validate the destination

The proposed name and URL are checked against existing presentations before any Webflow record is created or updated.

04Write the Webflow embed

The Webflow API creates or updates the CMS record and stores the shared iframe code in its presentation embed field.

05Create the full-screen page

The CMS entry supplies the iframe code to the custom Webflow template so the Vercel presentation fills the browser.

06Deploy and publish

The skill deploys the Webflow changes, publishes the CMS page, and returns the final prospect-ready URL.

One command, two publishing layers.

Vercel runs the presentation. Webflow gives the sales team a managed, branded URL they can put in front of a prospect without handling either deployment layer manually.
