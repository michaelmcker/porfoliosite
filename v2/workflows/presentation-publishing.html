Selected workflow · Publishing system

Presentation publishing

Moves branded HTML from an approved artifact to a prospect-ready Webflow URL without manual deployment or CMS handoff.

01 · Purpose

Why it exists

Custom HTML presentations can be more expressive than a standard slide deck, but deployment and CMS setup create a handoff between the finished artifact and the link a sales team can actually share. This publishing skill connects those layers while keeping the approved presentation intact.

02 · Sequence

How it works

The page and the artwork tell the same six-stage story, so the system remains understandable without relying on the raster diagram.

  1. 01

    Branded HTML presentation

    Start with a self-contained HTML artifact containing approved content and assets, plus a chosen name and destination.

  2. 02

    Vercel API deployment

    Publish the presentation into the corporate Vercel project.

  3. 03

    Name and destination validation

    Check the proposed name and URL against existing presentations before any CMS write.

  4. 04

    Webflow CMS iframe field

    Create or update the CMS record through the Webflow CMS API and write the shared iframe embed field.

  5. 05

    Full-screen presentation template

    The managed CMS page renders the Vercel-hosted artifact edge to edge.

  6. 06

    Published URL verification

    Open and verify both publishing layers in the browser, then return the final prospect-ready URL.

04 · Boundary

Public-safe constraints

For public review, credentials, tokens, and internal identifiers are intentionally omitted. This page documents the public-safe system boundary: what enters the workflow, how its two publishing layers connect, what a reviewer confirms, and what is returned.
